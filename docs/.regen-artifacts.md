# Regenerating committed example artifacts

Maintainer notes for rebuilding the tracked images under `examples/images/`.
Generated `.scad` files in `examples/` are gitignored build outputs — not
committed, nothing to rebuild for git.

## Morph animations (`*-assemble.apng`)

One command per morph. The BoxAndLid hero in `docs/morph.md`,
`docs/variants.md`, `docs/cheatsheet.md`, and `examples/README.md`:

```bash
.venv/bin/scadwright morph examples/box-and-lid.py assemble \
  examples/images/BoxAndLid-assemble.apng \
  --colorscheme Metallic \
  --openscad /Applications/OpenSCAD.app/Contents/MacOS/OpenSCAD
```

- `--colorscheme Metallic` gives the `#AAAAFF` lavender backdrop and gold
  shading that match the static beauty shots. Without it the background and
  material colors won't match the rest of the docs images.
- Defaults already match the committed file: `800x600`, `--frames 60`,
  `--fps 30`, `--loop`. Override only if a specific morph needs it.
- `--openscad` is required because the OpenSCAD binary isn't on PATH here.
  Alternatively export `SCADWRIGHT_OPENSCAD=/Applications/OpenSCAD.app/Contents/MacOS/OpenSCAD`.

## Static example shots (`*.png`)

Use the beauty-shot tool, which pins the shared camera, `Metallic`
colorscheme, and `#AAAAFF` background so every shot matches:

```bash
.venv/bin/python3 tools/render_beauty_shots.py --filter BoxAndLid
```

Drop `--filter` to rebuild all of them.
